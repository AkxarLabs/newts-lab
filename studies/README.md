# Studies

One directory per research effort: `studies/<slug>/`. This is the hub-side home for
**everything about one idea** — from first spark through to the finished paper. It is
created by `/ideate` and **grows** as the idea matures through the lifecycle; the
experiment **code** lives in a separate repo outside the hub (linked by the registry).

```
studies/<slug>/
├── IDEA.md          # frontmatter = state + scores; the idea itself (template: templates/idea/IDEA.md)
├── lit-review.md    # added by /lit-review
├── decisions.md     # added by /scope — ADR-style design deliberation record
├── proposal.md      # added by /propose (built from decisions.md); carries the Gate 1 signature
├── critiques/       # /critique-paper outputs for load-bearing external papers
├── reviews/         # proposal / paper review reports for this effort
├── sessions/        # /discuss collaborative-session docs for this effort (scope/in-project/target/paper)
└── paper/           # added by /write-paper (appears at the `writing` stage)
    ├── main.tex          # the paper (from templates/paper/ — venue chosen by writing.venue)
    ├── references.bib    # citations — only from this effort's lit-review notes
    ├── figures/          # generated by scripts in the project repo, synced here
    ├── claims.yaml       # every quantitative claim → run id + artifact path (audited by /review-paper)
    ├── artifacts/        # cited metrics.json archived at /finalize (hash-locked, audit-from-hub-alone)
    └── reviews/          # internal review reports
```

The slug is the effort's lab-wide ID — the same slug names its **project repo**
(`<lab.projects_root>/<slug>`, outside the hub) and its registry row. Most ideas die
before `writing`, so most `studies/<slug>/` dirs never grow a `paper/` — that's expected.

**The split this folder removes:** earlier the hub kept idea artifacts under `ideas/<slug>/`
and the paper under a separate top-level `papers/<slug>/`, joined only by the shared slug.
Consolidating them here means one `ls studies/<slug>/` shows the whole effort.

**The link to code stays in the registry.** The experiment code is its own git repo at
`<lab.projects_root>/<slug>`; `lab/REGISTRY.md`'s Project column is the single source of
truth for where it lives (no symlink — portable across machines, including Windows).
Hub tools (`audit_claims.py`, `sync_figures.py`, `lock_artifacts.py`) resolve it
registry-first, so an `/adopt`ed project at an arbitrary path still works.

Build a paper with `latexmk -pdf main.tex` inside `studies/<slug>/paper/` (LaTeX build
artifacts are gitignored).
