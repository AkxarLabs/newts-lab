# Lab Notebook

One entry per working session: `YYYY-MM-DD-<short-topic>.md`. Raw and chronological — what was done, what was observed, decisions made, surprises. Written at the end of every session (Hard rule 11).

Distill durable insights into `../knowledge/` — the notebook is the journal, knowledge is the world model.

Entry template:

```markdown
# YYYY-MM-DD — <topic>
**Context:** idea/project worked on, starting state.
**Did:** bullet list of actions (procedures run, experiments launched, with ids).
**Observed:** results, errors, surprises — with artifact pointers.
**Decided:** decisions + rationale (including "killed X because...").
**Next:** the single most important next action.
```
